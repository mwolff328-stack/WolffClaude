STEP 3 — Create phase4/core/high-elimination-detection.ts

Pure function. No I/O. No DB imports. Replace or rename phase4/core/chalk-collapse-detection.ts.

Export the detection function and the threshold constant:

  export const HIGH_ELIMINATION_THRESHOLD = 0.25;  // provisional

  export function detectHighEliminationWeeks(
    weeklyFieldConcentration: Phase1WeeklyFieldRow[],  // now includes result and eliminationContribution
    independentPathRows: Phase1IndependentPathRow[],
    remainingFieldSeries: Phase1ChartPoint[],
    poolSize: number,
  ): HighEliminationWeek[]

Logic:

1. Build fieldByWeek map from remainingFieldSeries. Week 0 = poolSize. Week N = survivors after week N.

2. For each week 1–18, compute weekEliminationRate:
   entering = fieldByWeek.get(week - 1) ?? poolSize
   after = fieldByWeek.get(week)
   rate = (entering - after) / entering

3. For each week where weekEliminationRate > HIGH_ELIMINATION_THRESHOLD:
   a. Collect all weeklyFieldConcentration rows for that week where result === 'L'
      Sort by eliminationContribution descending — these are topContributors
   b. explainedEliminationRate = sum of eliminationContribution across all losers
   c. unexplainedEliminationRate = weekEliminationRate - explainedEliminationRate
   d. Assign collapseType (PROVISIONAL):
      - CHALK_CONCENTRATION: any single loser has ownership >= 0.40 AND eliminationContribution >= 0.35
      - FRAGMENTED: no single loser has ownership >= 0.40
      - MIXED: multiple losers, at least one has ownership >= 0.30, but none satisfies both chalk thresholds
   e. Build chalkDetail only for CHALK_CONCENTRATION events:
      - Identify the chalk loser (ownership >= 0.40, eliminationContribution >= 0.35)
      - team, ownershipAtPickDeadline (= ownership), impliedWinProb from that row
      - independentPathPicks: filter independentPathRows to this week, this team, alive === true

4. Sort results by week ascending.
5. Return HighEliminationWeek[].
