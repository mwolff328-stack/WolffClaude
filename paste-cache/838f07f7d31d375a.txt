STEP 3 — Create phase4/core/high-elimination-detection.ts

Pure function. No I/O. No DB imports.

Replace phase4/core/chalk-collapse-detection.ts with this file (or rename and rewrite).

The function signature:

  export function detectHighEliminationWeeks(
    weeklyFieldConcentration: Phase1WeeklyFieldRow[],  // now includes result and eliminationContribution
    independentPathRows: Phase1IndependentPathRow[],
    remainingFieldSeries: Phase1ChartPoint[],
  ): HighEliminationWeek[]

Logic:

1. Build fieldByWeek map from remainingFieldSeries (week 0 = poolSize from meta, week N = survivors after week N)

2. Build weekEliminationRate for each week:
   rate = (fieldByWeek[week-1] - fieldByWeek[week]) / fieldByWeek[week-1]
   Week 1: entering = fieldByWeek[0]

3. For each week where weekEliminationRate > HIGH_ELIMINATION_THRESHOLD (provisional: 0.25):
   a. Collect all weeklyFieldConcentration rows for that week where result = 'L'
      These are the topContributors — sort by eliminationContribution descending
   b. Compute explainedEliminationRate = sum of eliminationContribution for all losers
   c. Compute unexplainedEliminationRate = weekEliminationRate - explainedEliminationRate
   d. Assign collapseType (PROVISIONAL thresholds):
      - CHALK_CONCENTRATION: any single loser has ownership >= 0.40 AND eliminationContribution >= 0.35
      - FRAGMENTED: no single loser has ownership >= 0.40
      - MIXED: multiple losers, at least one has ownership >= 0.30 but none crosses both chalk thresholds
   e. Build chalkDetail only for CHALK_CONCENTRATION:
      - team, ownershipAtPickDeadline, impliedWinProb from the chalk loser row
      - independentPathPicks: independentPathRows filtered to this week, this team, alive=true

4. Sort events by week ascending (deterministic)
5. Return events

Export HIGH_ELIMINATION_THRESHOLD = 0.25 as a named constant.
