DATA LOADING

Path B (default, no auth):
- On mount, call GET /api/games?season=2025 to retrieve the 18-week schedule and determine week date ranges
- Then call POST /api/pools/e6e25327-7755-4624-b9e8-7f9e67a974fd/portfolio/week for each of the 18 weeks, using { week: N, season: 2025, picksRequired: 1 }
- Cache all results in useSeasonData hook state — do not re-fetch during the session
- Show a loading state with progress (e.g., "Loading week 5 of 18...") while fetching

Path A (authenticated):
- Fetch pool config first to get picksRequired — derive it from pool config, do not hardcode
- Call POST /api/pools/{poolId}/portfolio/week for each week of the user's pool
- Auth uses connect.sid session cookie — include credentials: 'include' on all fetch calls
- If no session cookie is detected on load, redirect to https://survivorpulse.com/sign-in?redirect=<encoded current URL> (same tab, not new tab)

EDGE CASES — all must be handled:
- isPortfolioMode: false → show "Portfolio mode inactive — 1 entry remaining" in the week row, no survival delta
- 0 alive entries → show "Eliminated" row, gray out remaining weeks
- estimationMetadata.confidence < 0.5 → show data quality warning in Weekly Drill-Down
- Path A user has no entries → prompt to add entries before proceeding
- API call failure → inline error with retry button, do not crash session
- Bye week teams (winProbability: 0, opponent: "BYE") → filter out before displaying

SEASON REPLAY VIEW
- Season Summary Bar above the table: cumulative survival uplift, weeks diversified, weeks with correlated risk avoided. These are derived from loaded data — no additional API calls.
- Table: one row per week. Columns: Week (number + date range), Entries Alive, Coordinated Picks (Entry 1: TEAM, Entry 2: TEAM), Survival Delta (portfolioSurvivalDelta as % with +/- and color: green = positive, gray = 0, red = negative), Outcome (Win/Loss per entry, eliminated entries struck through)
- Click any row → opens WeeklyDrillDownModal for that week, increments weeksViewedCount in SessionContext
- Path B context card above table (only for Path B): "You're viewing Real 2025 Pool — a real 2025 NFL survivor pool with 2 alive entries. Here's how coordinated allocation would have played out across the full season."

WEEKLY DRILL-DOWN MODAL
Use shadcn Sheet or Dialog. Opens on row click, closes on X or outside click.

Header: Week number, date range, entries alive entering this week.

Allocation Table (one row per entry): Entry name, Assigned Team (name + opponent + spread), Win Probability (pWinNow as %), Yahoo Pick % (yahooPickPct as %), Overall Score (overallScore100), Rationale (perPickReasonByTeamId — display exactly as returned, no modification), Outcome (Win / Loss / blank if not played)

Portfolio Summary below allocation table:
- portfolioSurvivalDelta labeled "Survival uplift from coordination"
- coordinatedSurvivalProbability vs independentSurvivalProbability: "Coordinated: 48% | Independent: 36%"
- duplicateTeamsAvoided: "Teams avoided for overlap: DET, ARI"

Field Context below portfolio summary (from GET /api/pools/{poolId}/dynamics/comprehensive, called once per drill-down, cached by week):
- Chalk team: name, pick share %, win probability
- fieldSurvivalBaseline labeled "% of the field expected to survive this week"
- If estimationMetadata.confidence < 0.5: "Pick distribution estimates are based on limited historical data — treat with caution."
